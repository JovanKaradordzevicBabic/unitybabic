# gitHub_JovanBabic_vjezba_jedan

## Matej Tomrecaj 2tr1

## 1. Što je Visual Studio Code i za što se najčešće koristi?
Visual Studio Code je program za uređivanje koda, najčešće se koristi za programiranje, pisanje i uređivanje koda

## 2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
Zove se Explorer

## 3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
U lijevom izborniku odaberemo 5. ikonu Extensions i kliknemo gumb Install podred ekstenzije koju želimo instalirati.
Ekstenzije nam davaju puno više mogućnosti tijekom pisanja koda

## 4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
File -> Open file

## 5. Što se sve može raditi u Exploreru?
Možemo kopirati datoteke, brisati ih, gledati im strukturu

## 6. Kako možeš brzo pronaći određenu datoteku u projektu?
Možemo je pretražiti tako da pritisnemo CTRL + P i unesemo ime datoteke

## 7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
Pritisnemo CTRL + SHIFT + P i odaberemo split editor

## 8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
Pritisnemo CTRL + ALT + F

## 9. Što je version control i zašto se koristi?
Koristi se za praćenja promjena u kodu i omogućava nam ulazak u prethodne verzije koda

## 10. Kako se u VS Code-u otvara panel za Git (Source Control)?
U lijevom izborniku pritisnemo 3. ikonu grane

## 11. Što znače commit, push, pull?
Commit - lokalno spremi promjene
Push - šalje na GitHub
Pull - preuzima sa GitHub-a

## 12. Čemu služi commit poruka i kako treba izgledati?
Objašnjava što je korisnik napravio ili izmjenio
npr.
Add user login form
Fix navigation bug

## 13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
Kliknete na datoteku u Source Control panelu

## 14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
1. Spremi datoteku
2. Stage ('+' pored datoteke)
3. Commit (upiši poruku + kvačica)
4. Push (gumb ili '...' -> Push)

## 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
Desni klik na datoteku -> GitLens -> Timeline -> odaberi stari commit -> Revert

### 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
Otvoriti Source Control panel sa CTRL + SHIFT + G ili u terminalu: git status
